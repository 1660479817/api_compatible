## 1. 模块目标（摘要）

| 目标 | 说明 |
|------|------|
| 按 New API **用户** 分桶存全文 | 服务离线清洗、微调；元数据服务后续画像/分析 |
| 简单规则 + 全量 | 归属、脱敏、体积、保留期；**不** 在网关做清洗或 LLM |
| 插上即用 | 前置透明代理；不改 New API 源码 |
| 存储失败不挡推理 | `proxy-only` 旁路 |

**完整目标、非目标、存储长期契约**：见 [`DESIGN.md` §1–§2](../../experiment/corpus-tap/DESIGN.md)。

## 2. 画像 / 分析（扩展槽，非当前实现）

**Profile Analyzer** 与 **分析 Worker** 只读语料库，**不** 进入 Tap 热路径：

- 输入：`corpus-db` + S3（`http_exchange` 与 body 对象）  
- 输出：`user_profile` / `profiles/` 等（迁移 `003_profile.sql` 待建）  
- LLM：运营商独立 `CORPUS_PROFILE_LLM_*`，**禁止** 指向 New API 或用户 Bearer  

扩展槽目录与约束：[`DESIGN.md` §16](../../experiment/corpus-tap/DESIGN.md#16-扩展槽非本文实现范围)。

以下为 **画像期保留设计摘要**（实现画像时再展开为独立附录或 `profile/DESIGN.md`）：

| 原则 | 说明 |
|------|------|
| 与 Tap 隔离 | `corpus-profile` 独立容器；不 import `proxy` |
| 可关可开 | 未配分析 LLM 时可不启动 `corpus-profile` |
| 成本可控 | 默认抽样 + 每用户 LLM 调用上限 |

画像表结构、`profile_json` 字段建议、环境变量与内网 API：见本文件 Git 历史 §7.4–§8（2026-06-04 前版本）或待补 `profile/DESIGN.md`。

## 5. Compose 片段（运维）

生产 Tap 服务环境变量以 [`DESIGN.md` §13](../../experiment/corpus-tap/DESIGN.md#13-配置) 为准。最小片段：

```yaml
  corpus-tap:
    build: ../experiment/corpus-tap
    ports: ["8443:8443"]
    environment:
      CORPUS_TAP_UPSTREAM: http://new-api:3000
      CORPUS_TAP_DATABASE_URL: postgres://corpus:***@corpus-db:5432/corpus
      CORPUS_TAP_S3_BUCKET: corpus-raw-prototype
      CORPUS_TAP_NEWAPI_MYSQL_DSN: readonly://@mysql:3306/newapi
    depends_on: [new-api, corpus-db]

  corpus-db:
    image: postgres:16-alpine
```

可选 `corpus-profile`：见 §2，待画像阶段添加。

完整片段：[`experiment/corpus-tap/deploy/docker-compose.snippet.yml`](../../experiment/corpus-tap/deploy/docker-compose.snippet.yml)。

## 6. 出站与安全（G2 增量）

| 源 | 目标 |
|----|------|
| corpus-tap | new-api:3000、S3/MinIO、corpus-db、MySQL **只读**（`tokens`） |
| corpus-profile（可选） | corpus-db 只读、S3 读、`profiles/` 写、分析 LLM **独立** 443 白名单 |
